# Session: b-flow MVP Implementation

## Context
Built the b-flow XState orchestration extension for the Buck workflow Pi package. This is a durable state machine supervisor that delegates chunks to isolated worker subagents.

## Decisions Made
- **Direct implementation vs sub-agents**: `pi -p` sub-agent approach failed due to model timeouts and API key configuration issues. Implemented all 4 phases directly in the main context.
- **XState v5**: Used `setup()` API with typed guards/actors. Root-level `on` transitions use `.state` relative targets.
- **Child machine output**: `blockedChunk` transitions to `failed` final state with `output` containing block reason + queue. Parent reads output in `onDone`.
- **Immutable assign**: Replaced in-place mutation with `nextProjection()` returning new objects.
- **Persistence via exit actions**: Each state has `exit: persistAction()` instead of fragile root `always`.

## Gotchas
- XState v5 root-level `on` transitions need `.state` relative targets, not bare state names
- `node:test` in Node 24 doesn't export `expect`; use `node:assert` instead
- Pi ExtensionAPI `registerCommand` handler receives `args: string`, not `string[]`
- `session_before_compact` event return needs `{ compaction: { summary, firstKeptEntryId, tokensBefore } }`
- `setInterval` with captured `ctx` causes stale context errors after session replacement
