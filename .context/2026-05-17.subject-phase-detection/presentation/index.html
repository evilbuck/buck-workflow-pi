Implementation

The before_agent_start Hook

// Fires before every LLM turn — injects next work item
api.on("before_agent_start", async (event, ctx) => {
  const projection = readProjection(projectRoot);
  if (!projection || projection.currentState === "idle") return;

  // Find next work item (in-progress takes priority over pending)
  const nextItem = projection.queue.find(
    (q) => q.status === "in-progress" || q.status === "pending",
  );
  if (!nextItem || !existsSync(nextItem.path)) return;

  // Read just enough — 4KB snippet, no tool calls
  const snippet = readFileSync(nextItem.path, "utf-8").slice(0, 4000);

  const completed = projection.queue.filter(q => q.status === "completed").length;

  return {
    // Hidden message — LLM-only, not shown in TUI
    message: {
      customType: "b-workflow-next",
      content: `# Next Work — ${projection.subject}` + "\n" +
        `Queue: ${completed}/${projection.queue.length} done\n\n` + snippet,
      display: false,
    },
  };
});

Guard: idle state

If currentState === "idle", no injection. The XState machine's recovering → scanContext flow handles fresh starts.

Guard: missing file

If the next item's path doesn't exist on disk, silently skip. The XState machine will re-scan and rebuild the queue.

display: false

The message participates in LLM context but is hidden from the TUI. Clean chat, full context.

Queue Items

What the Queue Contains

📋 Phase

phase-N-slug.md

Discrete phase files with status: and difficulty: frontmatter. Ordered by filename. Scanned for first non-completed.

☑️ Task

tasks.md

Unchecked - [ ] items from the subject's tasks.md. Each unchecked line becomes a queue item.

📦 Backlog

backlog/todo.md

Linked backlog items from .context/backlog/todo.md. Each [title](items/slug.md) is a queue entry.

🔄 Iterate

iterate-*.md

Follow-up iteration bundles created during review cycles. Scanned from the subject folder.

Queue items are built by buildQueue() in queue-builder.ts during the executingChunks state. The order is: Phases → Tasks → Backlog → Iterate.
Command

/b-next — No LLM Needed

Quick inspection directly in the TUI. No tokens, no LLM turn, no tool calls.

Subject: 2026-05-15.refactor-auth
Path: .context/2026-05-15.refactor-auth/phase-2-migrate-auth.md
Queue: 1/4 done | State: executingChunks
api.registerCommand("b-next", {
  description: "Show next work item from b-flow queue",
  async handler(_args, ctx) {
    const projection = readProjection(projectRoot);
    if (!projection || projection.currentState === "idle") {
      ctx.ui.notify("No active b-flow session.", "info");
      return;
    }
    const current = inProgress ?? nextItem;
    if (!current) {
      ctx.ui.notify("Queue exhausted.", "info");
      return;
    }
    ctx.ui.notify(
      `▶️ Next: [${current.type}] ${current.id}\n` +
      `Queue: ${completed}/${total} done`,
      "info",
    );
  },
});

Compaction Hook

When context is compacted, the b-flow state summary is injected into the compaction entry. After compact, the before_agent_start hook re-injects the next item from the updated queue.

tool_result Hook

In extensions/index.ts, write/edit tool results are tracked into current-session.json. Memory files trigger QMD re-index. This feeds into b-save.
